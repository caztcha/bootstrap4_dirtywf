レスポンシブなカルーセル

複数パネルを表示する (画面サイズに応じてパネル数が増減する) カルーセルです (各パネルがリンクであるという想定)。jQuery プラグインの Slick を使用しています。

アクセシビリティの配慮

  • [Tab] キーによるフォーカス移動および [Enter] キーによる実行が可能。スクリーンリーダーを併用していない場合は、(フォーカスがカルーセル内にあれば) 左右矢印キーでもパネル切り替えが可能。
  • ボタン (前のパネル/次のパネル) の aria-label 属性値を日本語化 (slick.js を加筆修正)。
  • ボタン (前のパネル/次のパネル) の配色改善およびフォーカス非可視化 (outline:none;) の無効化 (slick-theme.css を加筆修正)。
  • ボタン (前のパネル/次のパネル) のタップ可能領域を拡大 (44 CSS pixels 四方)。また、ボタンとパネルのタップ可能領域の重なりを解決 (ボタンを優先) するために、ボタン側に z-index 値を追記。
  • 各パネルがリンクである場合、<a> 要素が親要素の下位にあると (たとえば、div や、箇条書きマークアップであれば li などの中に a があると)、Slick の仕様として、その親要素にもフォーカスが当たるようになっているため、スクリーンリーダーユーザーがリンクを利用できない (<a> 要素にフォーカスが当たってもラベルが読み上げられない) 恐れがある。このため、このサンプルでは、<a> が各パネルの親要素になるようにしている。
  • カルーセルの親要素 <div class="responsive-carousel"> に aria-live="polite" を記述。パネルを切り替えるたびにライブで、表示されるパネル (の画像 alt やテキストラベル) がスクリーンリーダーで読み上げられるようにしている。(Firefox + NVDA ではうまくいかず。)

固有の CSS および JavaScript

  • <head> 内の Additional CSS (for Slick) にある slick.css および slick-theme.css。
  • </body> 手前の Additional JavaScript (for Slick) にある slick.js およびカスタマイズのための script。
